## ❓ Frequently Asked Questions

**Q: Why can't simple binary search work here?**
A: Because array is not fully sorted - it's rotated. Parts are sorted, parts aren't.

**Q: How do we know which half is sorted?**
A: Check first and middle elements - one half is always sorted in rotated array.

**Q: What if there are duplicates?**
A: Approach becomes O(n) worst case. Problem 3 in Advanced covers this.

**Q: Is the O(log n) solution always possible?**
A: Yes! Using logic: one half is always sorted, other half must contain target if not in sorted half.

**Q: How do I remember this algorithm?**
A: Key insight: identify sorted half, check if target is in sorted half, recurse on appropriate half.

**Q: How do I prepare for interviews?**
A: Follow complete path + practice variations + explain clearly.

## 🌟 Key Concept

**The Core Insight:**
```
In a rotated sorted array, ONE HALF is always sorted!

For [3,4,5,6,7,0,1,2] with mid=6:
- Left half [3,4,5,6] is sorted ✓
- Right half [7,0,1,2] is NOT sorted

Use this to:
1. Identify which half is sorted
2. Check if target is in sorted half
3. Eliminate one half and recurse
= O(log n) solution!
```
